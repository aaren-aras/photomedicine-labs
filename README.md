# PhotoMedicine Labs

This repo contains some code for an end-to-end deep learning pipeline for retinal microvascular restoration and segmentation, developed as part of an **Undergraduate Research Assistantship** with PhotoMedicine Labs. It uses a 2-stage ResU-Net architecture and includes a desktop app made for 
targeted OCTA scanning via galvo-mirror region-of-interest (ROI) selection. Reproducible scripts 
for motion artifact simulation, dataset augmentation, and structural evaluation are also provided.

---

## Quick Start

```bash
# Install dependencies
pip install PyQt5 numpy matplotlib opencv-python tensorflow-cpu==2.17.0 \
            tf-keras imgaug scikit-learn tqdm

# 1. Preprocess datasets → .npy arrays
python data.py

# 2. Train (GPU, inside Docker)
docker compose up --build

# 3. Evaluate on test set
python evaluate.py

# 4. Launch GUI
python gui/oct_roi_selector.py
python gui/oct_roi_selector.py --image path/to/octa.png
```

## Background

**OCT** uses low-coherence near-infrared interferometry to produce depth-resolved cross-sectional B-scans of tissue at 2–10 µm axial resolution. **OCTA** extends this by comparing decorrelation between repeated B-scans at the same position — moving red blood cells produce decorrelated signal, static tissue doesn't. This difference is the angiogram.

SNR improves as √NR where NR = number of B-scan repeats. Standard clinical OCTA requires 4–12 repeats (14–21 seconds). Using only 2 repeats cuts acquisition time by 83% but severely degrades image quality. Stage 1 of this pipeline addresses that tradeoff.

## Data Pipeline (`data.py`)

Run once before training. Loads all raw images, applies CLAHE, resizes to 400×400, splits train/valid/test, saves as `.npy`. `np.load()` is 3–5× faster per batch than `PIL.Image.open()` and preserves exact uint8 values without compression artifacts.

### Datasets

| Source | Images | Resolution | Scanner | Split |
|--------|--------|-----------|---------|-------|
| OCTA-500 | 500 | 400×400 | Various retinal OCTA | 80/10/10 |
| ROSE-1 SVC+DVC | 117 | 304→400 px | Optovue SD-OCT | 80/20 |
| ROSE-2 | 112 | 840→400 px | Heidelberg SD-OCT | 80/20 |
| **Total** | **729** | 400×400 | 3 scanners | — |

OCTA-500 alone produced val_loss instability — the model memorised its specific noise characteristics. Adding ROSE across two additional scanners forces the model to learn general vessel features. The held-out test set is **OCTA-500 only** as a clean benchmark.

### Preprocessing steps

1. **Grayscale + resize** — PIL `convert('L')`, LANCZOS resampling. ROSE-1 upscaled, ROSE-2 downscaled, OCTA-500 kept native.
2. **CLAHE** — `clipLimit=2.0`, `tileGridSize=(8,8)`. Divides each image into 64 tiles of 50×50 px, equalises histogram locally per tile, bilinearly interpolates boundaries. Enhances vessel edge contrast without amplifying background noise. Validated for retinal OCTA by Giarratano et al. 2020.
3. **Binarize masks** — `(mask > 0).astype(uint8)` maps `[0,255]` → `{0,1}`. Required for Dice loss and BinaryCrossentropy.
4. **Save as `.npy`** — images uint8 `[0,255]`, masks uint8 `{0,1}`. Normalisation to float32 `[0,1]` happens inside the generator each epoch.

## Stage 1 — Image Restoration (`IRU-Net`)

Motivated by [Liao et al. 2023](https://doi.org/10.1364/BOE.486933) (Biomed. Opt. Express 14(8):3899). Stage 1 learns to reconstruct high-quality OCTA from degraded sparse acquisitions: **degraded OCTA → clean OCTA**.

### Architecture

ResU-Net encoder-decoder, 4 stages: 400→200→100→50 px. Filters: 16→32→64→128. Residual blocks **without BatchNorm** — Liao §3.1 found BN distorts intensity statistics in pixel reconstruction tasks, producing a washed-out appearance. Skip connections re-inject spatial detail from the encoder into the decoder at each resolution.

### Loss function

```
L_c = L2 + 0.01 × L_content
```

- **L2 (MSE)** stabilises training and suppresses noise. Alone it produces blurry outputs.
- **VGG19 content loss** compares deep feature maps at `block5_conv4` — the deepest layer, capturing vascular structure and texture rather than raw pixel values. Computed **before ReLU** (Liao §3.2) to avoid activation function interference. β=0.01 is optimal per Liao Table 1 ablation across β ∈ {0.001, 0.01, 0.1, 1.0}. VGG19 is frozen (`trainable=False`), grayscale tiled to RGB for 3-channel input, scaled to [0,255].

### Degradation simulation (on-the-fly)

Applied inside `restoration_generator()`, re-sampled every epoch so the model sees different noise realisations — implicit regularisation.

```
clean image
  → flip/rotate (geometric augmentation)
  → simulate_motion_artifacts()     # operates on augmented clean
  → add Gaussian noise σ=0.4
  → degraded image
```

Geometric augmentation is applied to the clean image **before** degradation so both clean and degraded automatically share the same orientation — no need to augment them independently.

Three physical artifact types:

| Artifact | Physical source | Simulation |
|----------|----------------|-----------|
| Shot noise | Photon counting fluctuations (Poisson) | Gaussian, σ ∝ √intensity |
| B-scan dropout | Eye saccades between repeat acquisitions | Random horizontal lines attenuated to 40% of neighbour |
| Rayleigh speckle | Coherent interference inherent to OCT | Multiplicative, Rayleigh distribution |

Gaussian noise σ=0.4 added after physics simulation (Liao §4.1) — simulates photon detector shot noise and improves generalisation to unseen noise levels.

### Metrics

| Metric | Description | Our result | Liao IRU-Net | Degraded input |
|--------|-------------|-----------|--------------|----------------|
| PSNR | Pixel fidelity (dB), higher = better | ~20–21 dB | 24.2 dB | 15.7 dB |
| SSIM | Perceptual quality [0,1], higher = better | ~0.79 | 0.59 | 0.28 |

Both well above the degraded input baseline. Gap vs Liao is expected — they trained on real 2-repeat vs 12-repeat paired data from their own SS-OCT system. We simulate degradation on clean OCTA-500 images. **The path to closing this gap is Andrei's microfluidic channel paired data** → retrain on real acquisition noise.

## Stage 2 — Vessel Segmentation (`SegU-Net`)

Binary vessel segmentation from OCTA projections. Adapted from the Oncer brain tumour segmentation project (BraTS 2021, 4-modality MRI ResU-Net). If `benchmark/restoration_best.keras` exists, Stage 1 is loaded frozen and images are degraded→restored before segmentation training — completing the end-to-end pipeline.

### Architecture

Same ResU-Net structure as Stage 1 but **with BatchNorm** — segmentation only needs to localise vessels, not reconstruct exact pixel intensities. BN improves training stability here without the reconstruction accuracy concern. Output: `Conv2D(1) + sigmoid` → vessel probability map `[0,1]`, thresholded at 0.5 for binary mask.

### Loss function

```
L = Dice + 0.3 × BCE
```

Vessels are ~10% of pixels. A model predicting all background achieves 90% accuracy but Dice=0. **Dice loss** directly optimises overlap and handles class imbalance by ignoring true negatives. **BCE** provides gradient signal at flat regions where Dice gradients vanish. Weight 0.3 lets Dice dominate — previous weight 0.5 over-penalised individual pixel errors and produced blurry vessel boundaries.

### clDice

Standard Dice treats all vessel pixels equally. **clDice (Shit et al., CVPR 2021)** additionally penalises breaks in vessel centerlines — critical for OCTA where connectivity determines FAZ boundary measurements and capillary density. Approximated via differentiable soft skeleton (iterated max pooling + subtraction, 5 iterations). `clDice = Dice = 0.873` in our results → vessel topology fully preserved, no centerline breaks.

### Val loss fix (was >1.0)

Four compounding issues resolved simultaneously:

| Fix | Effect |
|-----|--------|
| BCE weight 0.5 → 0.3 | Loss function was fighting itself |
| LR 5e-4 → 1e-4 | Optimizer was overshooting minima |
| Batch size 16 → 4 | GPU memory pressure + too few gradient updates/epoch |
| Added ROSE dataset | Model was overfitting OCTA-500 scanner characteristics |

### 3 generator bugs fixed

1. **Double path** — `np.load(img_dir / img_files[j])` prepended the directory twice (img_files already contains full Path objects from `.iterdir()`).
2. **imgaug shape** — `SegmentationMapsOnImage` needs `img.shape` (H,W,C) not `img.shape[:2]` (H,W).
3. **No epoch shuffle** — model memorised sample ordering. Fix: `random.shuffle(indices)` at the start of each epoch.

### Pipeline connection

When `restoration_best.keras` exists, `segmentation_generator()` loads it frozen and uses a direct Keras graph call rather than `model.predict()` to avoid per-call overhead inside the training loop:

```python
# Direct graph call — ~3-5x faster than model.predict() in a tight loop
restored_batch = restoration_model(degraded_batch, training=False).numpy()
```

The whole batch is restored in one call for GPU parallelism, rather than one image at a time.

## Results

### Segmentation — OCTA-500 held-out test set (30 images)

| Dice | IoU | Precision | Recall | Specificity | clDice |
|------|-----|-----------|--------|-------------|--------|
| **0.873** | **0.775** | 0.856 | 0.887 | 0.985 | **0.873** |

Giarratano 2020 benchmark (OCTA-500 large vessel): Dice 0.80–0.85. We exceed it. Precision < Recall (0.856 vs 0.887) → slight over-segmentation visible as small red FP patches in the overlay. clDice = Dice = 0.873 → no topological errors.

### Training configuration

| Parameter | Value | Source |
|-----------|-------|--------|
| Learning rate | 1×10⁻⁴ | Liao §4.1 |
| Adam β1 / β2 | 0.8 / 0.999 | Liao §4.1 — β1=0.8 forgets gradient history faster for sparse vessel gradients |
| LR decay | ×0.95 every 10k steps | Liao §4.1 |
| Batch size | 4 | Liao §4.1 — safe with VGG19 in Stage 1 loss |
| Max epochs | 400 | Liao §4.1 — early stopping terminates earlier |
| Early stop patience | 20 epochs | Liao §4.1 |
| Gaussian noise σ | 0.4 (restoration only) | Liao §4.1 |
| BCE weight | 0.3 | Empirical — 0.5 produced blurry masks |
| GPU | NVIDIA RTX 5070 Ti 16 GB | NGC TF 25.02 Docker |

> **Note on transferring Liao's hyperparameters:** β=0.01 and `block5_conv4` were ablated on skin OCTA. Optimizer settings (LR, β1, β2, decay) transfer well because they address properties of the OCTA signal itself rather than skin-specific characteristics. The content loss configuration would ideally be re-ablated on retinal OCTA.

## GUI (`gui/`)

PyQt5 desktop app bridging the segmentation pipeline with Andrei's OCT galvo hardware. Loads any OCTA image, drag-to-draw ROI, instantly computes galvo voltage sub-ranges. Segmentation runs **locally on CPU** from the saved `.keras` file — no Docker, no GPU required. ~1 second inference per image.

### Galvo voltage mapping

Andrei's OCT system drives X/Y galvo mirrors with voltage waveforms. Full FOV = X: 0→2V, Y: 0→2V. An ROI pixel sub-region maps proportionally:

```
voltage = v_min + (px / img_size) × (v_max - v_min)
```

Pixels 100–300 on a 500 px image → 0.4V → 1.2V on the X galvo. V_max is configurable via spin box. Voltage readout updates live on every mouse drag.

### Waveform generation

- **X fast axis — Sawtooth**: linear ramp `v_start→v_end` over `samples_per_line` samples, tiled `n_lines` times. Fly-back not included — DAQ handles retrace.
- **Y slow axis — Stair-step**: `n_lines` voltage steps linearly spaced `v_start→v_end`, each held for `samples_per_line` samples.

Together they define a raster scan over the ROI voltage sub-range. Export as `.npy` (numpy dict) or `.csv` for direct DAQ integration.

### Module architecture

| File | Responsibility |
|------|---------------|
| `constants.py` | Galvo V range, `MODEL_INPUT_SIZE=400`, `SEG_MODEL_PATH`, theme colours |
| `waveforms.py` | Pure math: `px_to_volts()`, `generate_sawtooth()`, `generate_stair_step()`, `roi_to_voltages()`. No Qt imports — fully testable in isolation. |
| `segmentation.py` | `SegmentationModel` wrapper. Lazy-loads on first `predict()`. Handles grayscale/RGB/RGBA input, resizes to 400×400, returns binary mask + RGBA overlay. |
| `roi_canvas.py` | `ROICanvas(QLabel)`. Mouse events, label↔image px coordinate mapping, overlay compositing, multi-ROI accumulation. |
| `waveform_canvas.py` | Matplotlib `FigureCanvas`. Updates on every ROI change. Preview capped at 50 lines for speed; full waveform generated at export time. |
| `theme.py` | Full QSS dark scientific stylesheet, applied once at startup. |
| `main_window.py` | UI wiring only. Centre panel built before left panel — `self.canvas` must exist before left panel references it. |
| `oct_roi_selector.py` | Entry point. `--image` arg for preloaded startup. |

## Infrastructure

### Docker GPU training

NVIDIA NGC TF 25.02-tf2-py3 container on RTX 5070 Ti (compute capability 12.0). NGC containers pre-package CUDA/cuDNN — eliminates DLL compatibility issues on Windows. Incremental GPU memory allocation (`set_memory_growth`) leaves headroom for VGG19 in Stage 1 loss.

```bash
docker compose up --build   # builds image + trains Stage 1 then Stage 2
```

`.dockerignore` excludes `data/` and `results/` — prevents ~3.6 GB of `.npy` files being copied on every rebuild, reducing `COPY . .` from ~170 seconds to ~2 seconds.

### Model conversion (.keras → .h5)

The `.keras` format saved inside the NGC container uses `tf_keras` serialisation. Convert to `.h5` (HDF5, version-agnostic) inside Docker for local inference:

```bash
docker compose run --rm photomedicine_labs python -c "
import tf_keras as keras
m = keras.models.load_model('benchmark/segmentation_best.keras', compile=False)
m.save('benchmark/segmentation_best.h5')
"
```

Then update `gui/constants.py`: `SEG_MODEL_PATH = MODELS_DIR / 'segmentation_best.h5'`

### TensorBoard

```bash
# Inside Docker
tensorboard --logdir benchmark/logs --host 0.0.0.0
# Or export static PNGs
python plot_training.py
```

Logs written to `benchmark/logs/restoration/` and `benchmark/logs/segmentation/`. Use `tf.compat.v1.train.summary_iterator()` (not the deprecated `tf_record_iterator`) with `import tensorflow as tf` at the top of `plot_training.py`.

### check.py

Pre-training assertions: shape == (400,400), dtype == uint8, image values in [0,255], mask values in {0,1}, vessel ratio 5–30%. Saves `raw_data_check.png` and `artifact_check.png` for visual inspection before committing to a full training run.
